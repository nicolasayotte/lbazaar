## F-05: NFT Certificate Reward — Teacher Configuration | MUST — Partial (~50%)

### F-05.1: Teacher enables NFT certificate — PARTIAL

**Backend (implemented):**
- `certificate_enabled` boolean field added to `courses` and `course_applications` tables (migration `2026_02_16_000001`)
- `certificate_status`, `certificate_tx_hash`, `certificate_minted_at` fields on `course_histories`
- `CertificateService` (`app/Services/API/CertificateService.php`) handles certificate metadata creation and on-chain minting
- Certificate minting API endpoints exist (see F-10)

**Frontend (missing):**
- Course Create/Edit form (`resources/js/pages/Portal/Course/Create.jsx`) has **no UI** for:
  - Toggle to enable/disable certificate reward
  - Certificate name/title input
  - Certificate image upload
  - Certificate description field
- Form still uses old `nft_id` mechanism instead of new `certificate_enabled` flag

**Tests:**
- `tests/Feature/CertificateEnabledFieldsTest.php` — validates database schema only
- No form submission or UI interaction tests

### F-05.2: Teacher disables NFT certificate — NOT VERIFIED

- No toggle exists in UI to disable; behavior cannot be triggered

### F-05.3: Teacher edits certificate config (enrollment-time locking) — NOT IMPLEMENTED

- SC-E02-06 requires reward config locked at enrollment time
- No snapshotting or versioning mechanism found
- No `enrollment_reward_config` column or similar on `course_histories`

### F-05.4: Incomplete metadata validation — NOT VERIFIED

- Backend validation exists in `CertificateFormRequestsIntegrationTest.php` for API-level requests
- No teacher-facing form validation since form doesn't exist

## F-10: Reward Airdrop — Teacher/Admin Initiated | MUST — Partial (~55%)

### F-10.1: Batch airdrop to all eligible students — PARTIAL

**Backend (implemented):**
- `POST /api/certificates/mint-and-airdrop` — batch airdrop endpoint
- `POST /api/certificates/courses/{course}/batch-mint` — batch mint for specific course
- `CertificateService` identifies eligible students (completed course, passed exams)
- Wallet priority rule implemented: external wallet preferred, fallback to custodial
- On-chain minting via Node.js scripts (`web3/run/build-certificate-tx.mjs`, `web3/run/submit-certificate-tx.mjs`)
- Transaction recorded in `nft_transactions` table

**Frontend (partially implemented):**
- `Certificates.jsx` component with batch mint button
- `CertificateTable.jsx` showing student certificate status

**Gaps:**
- **No fee estimation display** — spec requires showing total ADA cost (minting + network fees) before confirmation; not implemented
- **No Teacher wallet balance check** — no comparison of fee estimate vs wallet balance

### F-10.2: Airdrop to selected students — PARTIAL

- Backend supports single-student minting (`POST /api/certificates/courses/{course}/students/{student}/mint`)
- No multi-select UI for teacher to pick a subset of eligible students

### F-10.3: Admin initiates airdrop — NOT IMPLEMENTED

- No admin-specific routes or UI for admin-initiated airdrops
- No platform wallet integration for admin-paid fees

### F-10.4: No eligible students — NOT VERIFIED

- `GET /api/certificates/courses/{course}/eligible-students` returns list; empty list handling not verified in UI

### F-10.5: Insufficient ADA for fees — NOT IMPLEMENTED

- No wallet balance check before airdrop submission
- No shortfall messaging

### F-10.6: Partial airdrop failure — PARTIAL

- Backend records individual success/failure per student
- No retry-specific-students UI for failed deliveries

### F-10.7: Complete airdrop failure — PARTIAL

- Backend returns error; minimal error messaging in UI

### F-10.8: Teacher views class roster — NOT IMPLEMENTED

- No dedicated teacher roster view showing: completion status + reward delivery status + airdrop action
- Student wallet details correctly not exposed (spec-compliant)

### F-10.9: Duplicate airdrop prevention — DONE

- `CertificateService` checks `certificate_status` before minting; skips already-rewarded students (via airdrop or self-mint)
- Compliant with SC-E02-02

### F-10.10: No rewards configured — NOT VERIFIED

- Behavior when class has no rewards: airdrop columns and actions should be hidden from roster

### F-10.11: Teacher wallet not connected — NOT IMPLEMENTED

- No wallet connection validation before allowing airdrop initiation
- No prompt to connect wallet

## Cross-Cutting Gaps

1. **Token reward system entirely missing** — F-06 is the critical blocker for F-07, F-18.2, F-16.3, and token display in F-17.
2. **Teacher reward configuration UI** — No form fields in Course Create/Edit for either NFT or token configuration.
3. **Fee estimation** — Spec requires fee estimate display before every airdrop and self-mint; not implemented anywhere.
4. **Wallet connection validation** — Neither teacher (airdrop) nor student (self-mint) flows validate wallet connection before allowing action.
5. **Enrollment-time config locking (SC-E02-06)** — No mechanism to snapshot reward configuration at enrollment.
6. **Notification system** — No emails, no in-app badges, no eligibility prompts.
7. **Admin airdrop path** — No admin-specific routes, UI, or platform-wallet fee handling.
